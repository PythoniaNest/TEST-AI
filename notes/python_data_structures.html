Python Data Structures

Python provides several built-in data structures that are optimized for different types of operations. Understanding these data structures is crucial for writing efficient and maintainable code.

1. Lists

Lists are ordered, mutable sequences of elements. They can contain elements of different types.

Time Complexity

  • Access: O(1)
  • Insert/Delete at end: O(1)
  • Insert/Delete at beginning: O(n)
  • Search: O(n)

# Creating and using lists
fruits = ['apple', 'banana', 'orange']
numbers = [1, 2, 3, 4, 5]

# Common list operations
fruits.append('grape')    # Add element
fruits.pop()             # Remove last element
fruits.insert(1, 'kiwi') # Insert at position
fruits.remove('banana')   # Remove specific element

2. Tuples

Tuples are immutable sequences, typically used to store collections of heterogeneous data. Since tuples are immutable, they can be used as dictionary keys and are generally more memory efficient than lists.

Time Complexity

  • Access: O(1)
  • Search: O(n)
  • Hash (for dictionary keys): O(1)

# Creating and using tuples
coordinates = (10, 20)
person = ('John', 25, 'New York')

# Tuple operations
x, y = coordinates        # Unpacking
name, age, city = person  # Multiple assignment

3. Dictionaries

Dictionaries are mutable mappings of key-value pairs. They provide fast access to values through unique keys and are implemented using hash tables.

Time Complexity

  • Access: O(1) average case
  • Insert: O(1) average case
  • Delete: O(1) average case
  • Search by key: O(1) average case
  • Search by value: O(n)
Note: Dictionary performance can degrade to O(n) in worst-case scenarios due to hash collisions. 

# Creating and using dictionaries
student = {
    'name': 'Alice',
    'age': 20,
    'grades': {'math': 90, 'science': 85}
}

# Dictionary operations
student['course'] = 'Python'  # Add new key-value
del student['age']           # Remove key-value
print(student.get('name'))   # Safe access

4. Sets

Sets are unordered collections of unique elements. They are implemented using hash tables and are optimized for fast membership testing.

Time Complexity

  • Add: O(1) average case
  • Remove: O(1) average case
  • Membership testing: O(1) average case
  • Union/Intersection: O(min(len(s), len(t))) where s,t are sets
Note: Like dictionaries, set operations can degrade to O(n) in worst-case scenarios. 

# Creating and using sets
numbers = {1, 2, 3, 4, 5}
more_numbers = {4, 5, 6, 7, 8}

# Set operations
union = numbers | more_numbers
intersection = numbers & more_numbers
difference = numbers - more_numbers

Advanced Topics

Collections Module

Python's collections module provides additional data structures:

  • defaultdict: Dictionary subclass with default values
  • Counter: Dict subclass for counting hashable objects
  • deque: Double-ended queue with fast appends and pops
  • namedtuple: Tuple subclass with named fields

from collections import defaultdict, Counter, deque
# defaultdict example
word_count = defaultdict(int)
for word in ['apple', 'banana', 'apple', 'cherry']:
    word_count[word] += 1

# Counter example
inventory = Counter(['apple', 'banana', 'apple', 'cherry'])

# deque example
queue = deque(['first', 'second'])
queue.appendleft('new_first')
queue.append('last')
